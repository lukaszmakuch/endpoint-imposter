Endpoint Imposter

Declarative, stateful, fast, and reusable HTTP mocks.

Source code Documentation
Declarative statefulness

Real endpoints don't always return the same result for the same request. With scenarios, you can model such behavior in a declarative manner.

Control over when responses are sent back

No more brittle "delay 500ms"! The tests may tell the mocked sever when to finish a request the application started.

Reusable test doubles

Test doubles are data that lives in JavaScript modules. You can reuse them like any other JavaScript module. You can transform them like any other data.

Scalable mental model

The way you build mocks is the same regardless if you're testing just a small function or the whole application.

Fast parallelism

No need to worry about the state leaking between tests. A suffix added the root URL gives you an instantaneous, isolated session.

Platform agnostic

The mocks run as an independent HTTP server. Anything that works with HTTP will work with these mocks.
